# 違規統計功能測試指南

## 快速測試步驟

### 測試 1: 基本顯示測試

1. 打開 `index.html`
2. 檢查數字選擇器（A-K）右側是否出現三個統計卡片
3. 確認三個卡片標題分別為：
   - 訊號牌違規
   - 連續5局4張
   - 連續7局莊閒
4. 確認所有數字顯示為 0（綠色）

**預期結果**: ✅ 三個卡片正常顯示，數字為 0，顏色為綠色

---

### 測試 2: 生成牌靴測試

1. 選擇花色（例如：♥ 愛心）
2. 選擇數字（例如：10, J, Q, K）
3. 點擊 GO 按鈕生成牌靴
4. 觀察違規統計是否更新

**預期結果**: ✅ 生成完成後，違規統計自動更新

---

### 測試 3: 訊號牌違規檢測

**製造違規場景**:
1. 生成一個包含訊號牌的牌靴
2. 找到一個有訊號牌的局
3. 檢查下一局是否為「莊」
4. 如果不是，違規數應該 +1

**測試步驟**:
```
局數  |  有訊號牌  |  下一局結果  |  是否違規
 5    |    是     |     閒      |    是 ✗
 10   |    是     |     莊      |    否 ✓
 15   |    是     |     和      |    是 ✗
```

**預期結果**: ✅ 訊號牌違規數正確統計

---

### 測試 4: 連續4張牌檢測

**製造違規場景**:
1. 找到連續 5 局或以上都是 4 張牌的區段
2. 違規數應該顯示該區段的數量

**例如**:
```
局1: 4張 ─┐
局2: 4張  │
局3: 4張  ├─ 連續 5 局都是 4 張（違規！）
局4: 4張  │
局5: 4張 ─┘
局6: 5張 ← 打斷連續
```

**預期結果**: ✅ 連續5局4張違規數正確顯示為 1

---

### 測試 5: 連續7局莊閒檢測

**製造違規場景**:
1. 找到連續 7 局或以上都開莊或都開閒的區段
2. 違規數應該顯示該區段的數量

**例如**:
```
局1: 莊 ─┐
局2: 莊  │
局3: 莊  │
局4: 莊  ├─ 連續 7 局都是莊（違規！）
局5: 莊  │
局6: 莊  │
局7: 莊 ─┘
局8: 閒 ← 打斷連續
```

**注意**: 和局會被跳過
```
局1: 莊 ─┐
局2: 莊  │
局3: 和  │← 跳過，不打斷也不計數
局4: 莊  ├─ 實際連續 7 個莊（違規！）
局5: 莊  │
局6: 莊  │
局7: 莊  │
局8: 莊 ─┘
```

**預期結果**: ✅ 連續7局莊閒違規數正確顯示

---

- ✅ 修正前：訊號牌違規 = 3

- 動畫：呼吸效果（1.5秒循環）

**測試方法**:
1. 生成無違規牌靴 → 檢查綠色樣式
2. 生成有違規牌靴 → 檢查紅色樣式和動畫

**預期結果**: ✅ 樣式和動畫正確切換

## 常見問題排查

### 問題：違規統計沒有顯示

**排查步驟**:
1. 檢查瀏覽器控制台是否有錯誤
2. 確認 `signals.js` 和 `signals_ui.js` 已正確載入
3. 檢查 HTML 中是否有違規統計的 div 元素
4. 檢查 CSS 樣式是否正確載入

### 問題：點擊操作後違規數沒有更新

**排查步驟**:
1. 打開瀏覽器控制台
2. 輸入 `typeof refreshViolationStats` 檢查函數是否存在
3. 手動執行 `refreshViolationStats()` 測試
4. 檢查事件監聽器是否正確設置

### 問題：違規數顯示錯誤

**排查步驟**:
1. 在控制台輸入 `calculateViolationStats(currentRounds)` 查看原始計算結果
2. 檢查 `currentRounds` 資料結構
3. 確認違規檢測邏輯是否符合規則
